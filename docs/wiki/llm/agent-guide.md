# Guia para agentes e LLMs

## Como navegar a wiki
1. Comece pelo **hub**: `docs/wiki/README.md`.
2. Use **Índices avançados** para localizar o domínio (server/client/protocol).
3. Consulte **Mapa de Arquitetura** para contexto macro.
4. Use o **Glossário** para termos específicos.

## Onde encontrar respostas arquiteturais
- **Macro e camadas**: `docs/wiki/ARQUITETURA.md`.
- **Fluxos**: `docs/wiki/FLUXOS.md`.
- **Sistemas**: `docs/wiki/SISTEMAS.md` e `docs/wiki/game-systems/`.

## Onde encontrar exemplos
- **Servidor**: `docs/wiki/server/README.md`.
- **Cliente**: `docs/wiki/client/README.md`.
- **Scripts**: `docs/wiki/scripts/README.md`.

## Onde NÃO inferir comportamento
- Não deduzir opcodes ou fluxos não listados em `docs/wiki/protocol/README.md`.
- Não assumir scripts ou módulos que não existam no repositório.
- Não extrapolar comportamento além dos arquivos citados.

## Boas práticas de resposta
- Sempre citar arquivos referenciados.
- Priorizar trechos com exemplos reais.
- Evitar sínteses que introduzam novos sistemas.
