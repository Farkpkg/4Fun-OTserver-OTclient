# Compartilhado / apoio

## Visão geral
Esta seção cobre artefatos compartilhados do repositório que servem como apoio para o servidor e o cliente, principalmente documentação, referências e material de contexto.

## Papel no projeto
- Centralizar regras e referências de documentação.
- Armazenar materiais de apoio para execução e análise do repositório.

## Localização no repositório
- `docs/` — documentação geral (regras, playbooks, enums).
- `ai/` — contexto de execução e referências internas.
- `codex-client-readmes/` — documentação organizada por áreas do OTClient.

## Estrutura interna
- `docs/` contém arquivos Markdown e templates para processos internos.
- `ai/` concentra contexto de arquitetura e protocolos.
- `codex-client-readmes/` espelha áreas do cliente com guias específicos.

## Integrações
- **Servidor/Cliente**: não há integração de runtime; serve como base de documentação e referência.
